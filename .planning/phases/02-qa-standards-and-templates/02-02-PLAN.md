**Required Sections (in this order):**

1. **Title + Purpose block** -- "VALIDATION_REPORT.md Template", purpose: 4-layer validation results for generated test code, producer: qa-validator, consumers: qa-bug-detective + human reviewer.

2. **Section: Summary** -- Table with columns: Layer (Syntax/Structure/Dependencies/Logic), Status (PASS/FAIL), Issues Found (count), Issues Fixed (count). Also include: total files validated, total issues, fix loops used, overall status.

3. **Section: File Details** -- Per validated file: filename, then a table with Layer, Status, Details (what passed, what failed, specific error messages). Group by file. Show both passing and failing layers.

4. **Section: Unresolved Issues** -- Issues that could NOT be auto-fixed after 3 loops. Each: file path, layer, issue description, attempted fix, why it failed, suggested manual resolution. If no unresolved issues, state "None -- all issues resolved within 3 fix loops."

5. **Section: Fix Loop Log** -- Chronological log of each fix loop iteration. Each entry: Loop number (1/2/3), issues found this loop, fixes applied, verification result. Shows the progressive resolution of issues.

6. **Section: Confidence Level** -- Overall confidence: HIGH/MEDIUM/LOW with reasoning. Factors: percentage of layers passing, severity of unresolved issues, number of fix loops needed. Table mapping confidence to criteria:
   - HIGH: All layers PASS, 0 unresolved, max 1 fix loop
   - MEDIUM: All layers PASS after fixes, 0-2 minor unresolved
   - LOW: Any layer still FAIL, or critical unresolved issues

2. **Section: Summary** -- Table with columns: Classification (APPLICATION BUG / TEST CODE ERROR / ENVIRONMENT ISSUE / INCONCLUSIVE), Count, Auto-Fixed count, Needs Attention count. Also: total failures analyzed, total auto-fixed, total requiring human attention.

3. **Section: Detailed Analysis** -- Per failure, ALL of these fields are MANDATORY:
   - Test Name: the failing test case (with ID)
   - Classification: one of the 4 categories
   - Confidence: HIGH / MEDIUM-HIGH / MEDIUM / LOW
   - File: exact path:line number
   - Error Message: complete error text
   - Evidence: code snippet showing the issue + reasoning for classification
   - Action Taken: "Auto-fixed" or "Reported for human review"
   - Resolution: what was fixed OR what the human needs to investigate

   Include the classification decision tree reference: why THIS category and not another.

4. **Section: Auto-Fix Log** -- Table: failure ID, original error, fix applied, confidence at time of fix, verification result (PASS/FAIL after fix). Only TEST CODE ERROR with HIGH confidence gets auto-fixed.

5. **Section: Recommendations** -- Grouped by category:
   - APPLICATION BUG: which bugs to prioritize, suggested investigation steps
   - TEST CODE ERROR: patterns to improve (e.g., "3 tests had missing await -- add ESLint rule for no-floating-promises")
   - ENVIRONMENT ISSUE: environment setup improvements
   - INCONCLUSIVE: what additional information would help classify
