   2. Add missing P0 tests

**Required Sections (in this order):**

1. **Title + Purpose block** -- "QA_AUDIT_REPORT.md Template", purpose: comprehensive quality audit of an existing test suite across 6 dimensions, producer: qa-validator (audit mode), consumers: human reviewer + qa-executor (for fixes).

2. **Section: Executive Summary** -- Overall health score: 0-100 with letter grade (A: 90-100, B: 80-89, C: 70-79, D: 60-69, F: <60). One-paragraph assessment. Key strengths (2-3 bullets). Key weaknesses (2-3 bullets). Recommendation: "PASS -- ready for CI" or "NEEDS IMPROVEMENT -- address critical issues first."

3. **Section: 6-Dimension Scoring** -- Table: Dimension, Score (0-100), Grade, Weight, Weighted Score, Key Finding. The 6 dimensions:
   - **Locator Quality** (weight: 20%): % Tier 1+2 locators vs Tier 3+4. Deduct points for every Tier 4 locator without a TODO comment.
   - **Assertion Specificity** (weight: 20%): % concrete assertions (toBe, toEqual, toHaveText with values) vs vague (toBeTruthy, toBeDefined, should('exist')). Deduct for every vague assertion.
   - **POM Compliance** (weight: 15%): 6 POM rules checked per page object (one class per page, no assertions, locators as properties, actions return void/page, state queries return data, extends base).
   - **Test Coverage** (weight: 20%): Pyramid distribution match (actual vs recommended), module coverage completeness.
   - **Naming Convention** (weight: 15%): File names, test IDs, fixture names following standards.
   - **Test Data Management** (weight: 10%): No hardcoded credentials, fixtures in dedicated folder, env vars for config, each domain has own fixture file.

   Show formula: Overall Score = sum(dimension_score * weight). Show the calculation.

4. **Section: Critical Issues** -- Issues that MUST be fixed immediately (BLOCKER severity). Each: file path, line number, issue description, why it's critical, suggested fix. These block CI/delivery.

5. **Section: Improvement Recommendations** -- Prioritized list. Each: recommendation, dimension it improves, estimated effort (S/M/L), expected score impact (+N points). Grouped by effort (quick wins first).

6. **Section: Test File Inventory** -- Summary: total test files, total test cases, per-pyramid-tier breakdown (unit/integration/API/E2E counts). Table: file path, tier, test count, pass/fail, last modified.

7. **Section: Detailed Findings** -- Per test file: file path, then table: Line, Issue, Severity (BLOCKER/WARNING/INFO), Dimension (which of the 6), Description, Suggested Fix. Group by file.
