    **<process> section with 7 steps:**
    1. `read_inputs` (priority="first") -- Read TEST_INVENTORY.md and QA_ANALYSIS.md completely. Extract: all test case IDs with their targets and priorities, architecture type, framework, identified features/modules from analysis.
    2. `analyze_features` -- Extract feature domains from test case IDs and targets. Pattern: UT-AUTH-*, API-AUTH-*, E2E-LOGIN-* all belong to "auth" feature. Group by feature domain, not pyramid tier. Per CONTEXT.md locked decision: "Groups test files by feature (auth tests together: unit+API+E2E), not by pyramid level."
    3. `create_feature_groups` -- For each feature, collect all test cases across pyramid tiers. Each feature group contains: feature name, test case IDs (unit + integration + API + E2E), estimated file count, complexity assessment.
    4. `determine_dependencies` -- Identify task ordering within and across features. Rules: (a) BasePage before any feature POM, (b) Feature POMs before E2E specs that use them, (c) Fixtures before tests that reference them, (d) Config files before any test execution, (e) Shared utilities before consumers. Record dependencies as "task X must complete before task Y" pairs.
    5. `assign_files` -- Map test cases to output file paths following CLAUDE.md naming conventions. Unit: tests/unit/{feature}.unit.spec.{ext}. API: tests/api/{resource}.api.spec.{ext}. E2E: tests/e2e/smoke/{feature}.e2e.spec.{ext}. POM: pages/{feature}/{Feature}Page.{ext}. Fixtures: fixtures/{domain}-data.{ext}. Config: config files at root or config/. Extension matches detected framework.
    6. `produce_plan` -- Write the generation plan markdown with per-task structure: task_id, feature_group, files_to_create (paths), test_case_ids (which IDs from TEST_INVENTORY this task implements), depends_on (other task IDs), estimated_complexity (LOW/MEDIUM/HIGH). Include a summary section with total tasks, total files, dependency graph visualization.
    7. `validate_plan` -- Verify: (a) every test case ID from TEST_INVENTORY.md appears in exactly one task, (b) no test case is assigned to multiple tasks, (c) no test case is unassigned, (d) dependency graph is acyclic (no circular deps), (e) file paths follow naming conventions, (f) every feature group has at least one task.

    **<output> section:**
    - Generation plan markdown at orchestrator-specified path
    - Structured return: PLANNER_COMPLETE with file_path, total_tasks, total_files, feature_count, dependency_depth, test_case_count, commit_hash

    **<quality_gate> section:**
    From CLAUDE.md Verification Commands (verbatim):
    - [ ] Test cases mapped to output files
    - [ ] No unassigned cases
    - [ ] No duplicate assignments
    Additional planner-specific checks:
    - [ ] Groups organized by feature, not by pyramid level
    - [ ] Dependencies are acyclic
    - [ ] File paths follow CLAUDE.md naming conventions
    - [ ] Every feature group has unit + API or E2E coverage (not single-tier groups unless feature only warrants it)
    - [ ] BasePage listed as dependency for all POM tasks
    - [ ] Output path matches orchestrator specification

    **<success_criteria> section:**
    Numbered list: generation plan exists, all test cases assigned, dependencies acyclic, committed, return values provided, quality gate passes.
