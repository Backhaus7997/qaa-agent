       - Repeat up to 3 times maximum
       - After 3 loops with unresolved issues: CHECKPOINT_RETURN with remaining issues for user decision

       - Validator does NOT commit fixes -- leaves them in working tree

Calculate confidence: HIGH (all pass, 0 unresolved, 0-1 loops), MEDIUM (all pass after fixes, 0-2 minor unresolved, 2-3 loops), LOW (any layer still FAIL, or critical unresolved, 3 loops exhausted).
    8. `return_results` -- Return structured result to orchestrator: VALIDATOR_COMPLETE with report_path, overall_status (PASS/PASS_WITH_WARNINGS/FAIL), confidence (HIGH/MEDIUM/LOW), layers_summary (4 entries), fix_loops_used, issues_found, issues_fixed, unresolved_count. Do NOT commit.

    **<process> section with 8 steps:**
    1. `read_inputs` (priority="first") -- Read CLAUDE.md (classification rules, quality gates). Read templates/failure-classification.md (output format, 4 sections, decision tree). Read qa-bug-detective SKILL.md for classification decision tree and auto-fix rules.
    2. `detect_test_runner` -- Detect test framework from config files: playwright.config.* (Playwright), cypress.config.* (Cypress), jest.config.* (Jest), vitest.config.* (Vitest), pytest.ini/pyproject.toml (pytest). Also check package.json scripts for test commands. Claude's discretion on exact detection logic. If no test runner detected: CHECKPOINT_RETURN asking user which runner to use and how to invoke it.
    3. `run_tests` -- Execute the test suite using detected runner. Command examples: npx playwright test, npx cypress run, npx jest, npx vitest run, pytest. Capture stdout, stderr, exit code. Parse test results to get pass/fail per test case. Per CONTEXT.MD: "Actually runs the test suite (not static analysis). Captures real output, classifies real failures. Requires test environment."
    4. `classify_failures` -- For each failure, apply the decision tree from SKILL.md:
       - Syntax/import error in TEST file? -> TEST CODE ERROR
       - Error in PRODUCTION code path (src/, app/, lib/)? -> Check: known bug / unexpected behavior? -> APPLICATION BUG. Code works as designed but test expectation wrong? -> TEST CODE ERROR
       - Connection refused / timeout / missing env var? -> ENVIRONMENT ISSUE
       - Cannot determine? -> INCONCLUSIVE
       Per CONTEXT.MD: "Classification: APP BUG / TEST CODE ERROR / ENV ISSUE / INCONCLUSIVE with evidence and confidence levels"
    5. `collect_evidence` -- For each failure, gather ALL 6 mandatory evidence fields: (1) exact file:line, (2) complete error message (full text, not summary), (3) code snippet proving the classification, (4) confidence level (HIGH/MEDIUM-HIGH/MEDIUM/LOW), (5) reasoning explaining why THIS classification and not another, (6) action recommendation.
    6. `auto_fix` -- Per CONTEXT.MD and SKILL.md: auto-fix ONLY when: classification = TEST CODE ERROR AND confidence = HIGH. Fix types: import path corrections, selector updates (match current DOM/data-testid), assertion value updates, config fixes (baseURL, timeout), missing await keywords, fixture path corrections. Per CONTEXT.MD: "Never touches application code. Only modifies test files. Application bugs are always report-only." After each fix, VERIFY by re-running the specific failing test. Record fix result (PASS/FAIL).
    7. `produce_report` -- Write FAILURE_CLASSIFICATION_REPORT.md matching templates/failure-classification.md exactly (4 sections). Summary table must include ALL 4 categories even if count is 0. Detailed Analysis has per-failure entries with all mandatory fields. Auto-Fix Log documents every fix with verification result. Recommendations grouped by category and specific to failures found.
    8. `return_results` -- Return structured result: DETECTIVE_COMPLETE with report_path, total_failures, classification_breakdown (app_bug/test_error/env_issue/inconclusive counts), auto_fixes_applied, auto_fixes_verified, commit_hash.

    **Commit behavior:**
    Commit the report and any auto-fixed test files:
    node bin/qaa-tools.cjs commit "qa(bug-detective): classify {N} failures - {breakdown}" --files {report_path} {fixed_files}
