**The new deliver stage must implement these sub-steps in order:**

**Sub-step 1: Pre-flight checks**
- Check `git remote get-url origin` succeeds. If no remote, print error: "No git remote found. Artifacts committed locally but PR creation skipped." and fall back to local-only commit (skip push and PR creation).
- Check `gh auth status` succeeds. If not authenticated, print error: "gh CLI not authenticated. Run 'gh auth login' first. Artifacts committed locally." and fall back to local-only commit.

**Sub-step 2: Derive project name**
- Read `dev_repo_path` from init context (available from Step 1).
- Attempt: `node -e "try { const p = require('${dev_repo_path}/package.json'); console.log(p.name || ''); } catch { console.log(''); }"`
- Fallback: use directory basename of `dev_repo_path`
- Sanitize: lowercase, replace non-alphanumeric with hyphens, collapse multiple hyphens, trim leading/trailing hyphens.

**Sub-step 3: Detect default branch**
- Run: `gh repo view --json defaultBranchRef --jq '.defaultBranchRef.name'` with fallback to "main" if command fails.

**Sub-step 4: Create feature branch**
- Branch name: `qa/auto-{project_name}-{date}` (date from init context).
- Handle collision: if branch exists locally or remotely, append `-2`, `-3`, etc.
- Create from default branch: `git checkout -b {branch_name} {default_branch}`

**Sub-step 5: Per-stage atomic commits**
- For each pipeline stage that produced artifacts, commit using the EXACT message format already defined in the orchestrator:
  - Scanner: `qa(scanner): produce SCAN_MANIFEST.md for {project_name}` -- files: `{output_dir}/SCAN_MANIFEST.md`
  - Analyzer (Option 1): `qa(analyzer): produce QA_ANALYSIS.md and TEST_INVENTORY.md` -- files: `{output_dir}/QA_ANALYSIS.md {output_dir}/TEST_INVENTORY.md {output_dir}/QA_REPO_BLUEPRINT.md`
  - Analyzer (Option 2/3): `qa(analyzer): produce GAP_ANALYSIS.md` -- files: `{output_dir}/GAP_ANALYSIS.md`
  - TestID injector (if ran): `qa(testid-injector): inject {N} data-testid attributes across {M} components` -- files: `{output_dir}/TESTID_AUDIT_REPORT.md` plus modified source files
  - Executor: `qa(executor): generate {N} test files with POMs and fixtures` -- files: all generated test files
  - Validator: `qa(validator): validate generated tests - {status} with {confidence} confidence` -- files: `{output_dir}/VALIDATION_REPORT.md`
  - Bug detective (if ran): `qa(bug-detective): classify {N} failures - {breakdown}` -- files: `{output_dir}/FAILURE_CLASSIFICATION_REPORT.md`
- Use `node bin/qaa-tools.cjs commit "{message}" --files {file_paths}` for each.
- Skip stages that did not produce artifacts (check file existence before committing).

**Sub-step 6: Push branch**
- Run: `git push -u origin {branch_name}`
- If push fails, print error and continue (artifacts are committed locally).

**Sub-step 7: Build PR body**
- Read `templates/pr-template.md`.
- Replace all `{placeholder}` tokens with actual values collected during pipeline execution:
  - Architecture type and framework from QA_ANALYSIS.md (or init context)
  - Risk summary from QA_ANALYSIS.md risk assessment section
  - Test counts by pyramid level from TEST_INVENTORY.md summary
  - Coverage metrics from QA_ANALYSIS.md
  - Validation result, confidence, fix loops from VALIDATION_REPORT.md
  - File list: if total files <= 50, list each file as `- {path}`. If > 50, use summary: `{N} files across {M} directories`.
- The PR body must be in English (per locked decision).

**Sub-step 8: Create draft PR**
- Run:
```bash
gh pr create \
  --draft \
  --title "qa: automated test suite for {project_name}" \
  --body "{filled_pr_body}" \
```

- Do NOT pass `--base` flag (let gh auto-detect default branch -- avoids Pitfall 5).
- Capture and store the PR URL from stdout.
- If `gh pr create` fails, print error with the exact failure message. Artifacts remain on the branch.
