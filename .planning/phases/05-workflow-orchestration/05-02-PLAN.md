**2. `<checkpoint_system>` section** (~80 lines) for FLOW-07:
Define checkpoint classification and handling:

```markdown
## Checkpoint System

### Checkpoint Classification

Every agent may return checkpoint data. The orchestrator classifies each checkpoint as SAFE or RISKY.

**SAFE checkpoints (auto-approve in auto mode):**
| Checkpoint | Agent | Why Safe |
|------------|-------|----------|
| Framework detection uncertain (LOW confidence) | Scanner | Auto-select most likely framework |
| Analyzer assumptions review | Analyzer | Assumptions are informational |
| TestID audit review | TestID Injector | P0-only injection is conservative |

**RISKY checkpoints (ALWAYS pause, even in auto mode):**
| Checkpoint | Agent | Why Risky |
|------------|-------|-----------|
| Validator escalation (unresolved issues after 3 fix loops) | Validator | Unresolved issues mean tests may be broken |
| APPLICATION BUG classification | Bug Detective | Requires developer action |
| Any checkpoint with `blocking` containing "unresolved" or "failed" | Any | Indicates pipeline integrity risk |

### Checkpoint Handling Flow

```
On agent return with checkpoint data:
  1. Extract checkpoint `blocking` field content
  2. Classify as SAFE or RISKY:
     - Match against safe patterns: "framework detection", "assumptions", "audit", "data-testid"
     - Everything else defaults to RISKY
  3. If IS_AUTO and SAFE:
     - Auto-approve with default action
     - Log: "Auto-approved: {checkpoint_description}"
     - Continue pipeline
  4. If IS_AUTO and RISKY:
     - PAUSE pipeline
     - Print checkpoint details with full context
     - Wait for user input
     - On user response: spawn fresh continuation agent
  5. If NOT auto:
     - PAUSE pipeline
     - Print checkpoint details
     - Wait for user input
     - On user response: spawn fresh continuation agent
```

### Resume After Checkpoint

When resuming after a checkpoint, spawn a FRESH agent (not serialized state):

```
Task(
  prompt="
    <objective>Continue QA pipeline from {stage} stage</objective>
    <execution_context>@agents/qa-pipeline-orchestrator.md</execution_context>
    <resume_context>
    Pipeline state:
    - Completed stages: {list of completed stages}
    - Current stage: {stage that triggered checkpoint}
    - Checkpoint response: {user's response}
    - Artifacts produced so far: {list of files}

    Resume from: {exact point in pipeline}
    User decision: {what user chose at checkpoint}
    </resume_context>
  "
)
